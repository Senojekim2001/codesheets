# Ed25519 / RSA — public-key signing and verification

> Ed25519PrivateKey, RSAPrivateKey, sign / verify, PEM serialization, kid-based key rotation, JWT signing keys, X25519 key exchange

## Overview

Public-key signing lets anyone with the public key verify that a payload was signed by the holder of the matching private key. Ed25519 (EdDSA over Curve25519) is the default for new code: 32-byte keys, 64-byte signatures, no parameter choices to mess up, fast verify. RSA persists where libraries or specs demand it — JWT RS256, X.509 certificates, S/MIME, smartcards. Versus HMAC (previous entry): HMAC needs a SHARED secret (anyone who can verify can also sign); public-key signing separates the two roles. The three examples solve the SAME concrete task — sign a release artifact (or a JWT) so anyone with the public key can verify, with key rotation support — at three depths: literal Ed25519 generate/sign/verify → PEM serialization + JWT-style kid rotation + RSA for legacy → production with JWKS endpoint serving public keys, signing service via KMS, sigstore-style ephemeral keys.

## Task

All tiers below solve the SAME concrete task at increasing depth — the differences are sophistication, not subject:

- **Intro** — Sign a payload with a private key; verify with the public key. Wrong key or tampered data -> verify fails.
- **Junior** — SAME — but persist keys via PEM serialization, support multiple active key versions for rotation, fall back to RSA for legacy.
- **Senior** — SAME — production: serve public keys via a JWKS endpoint so verifiers can fetch + cache them; sign via KMS so the private key never lives in application memory; ephemeral keys for short-lived signatures (sigstore-style).

## Signature

```python
sk = Ed25519PrivateKey.generate(); sig = sk.sign(data); sk.public_key().verify(sig, data)
```

## Example — Intro (Entry-Level)

```python
# === ENTRY-LEVEL EXAMPLE ===
# TASK      - Sign a payload with a private key; verify with the
#             public key. Wrong key or tampered data -> verify fails.
# APPROACH  - Ed25519PrivateKey.generate(), .sign(), public.verify().
# STRENGTHS - Smallest correct asymmetric signing; no parameter choices
#             (key size, hash algorithm); 32-byte keys, 64-byte sigs.
# WEAKNESSES- In-memory keys only; junior tier shows PEM serialization
#             for storage.
from cryptography.hazmat.primitives.asymmetric.ed25519 import (
    Ed25519PrivateKey, Ed25519PublicKey,
)

# Generate once; in production load from disk or KMS.
private_key = Ed25519PrivateKey.generate()
public_key  = private_key.public_key()

# Sign.
def sign(data: bytes) -> bytes:
    return private_key.sign(data)                     # 64 bytes

# Verify (raises InvalidSignature on mismatch).
def verify(public: Ed25519PublicKey, sig: bytes, data: bytes) -> bool:
    try:
        public.verify(sig, data)
        return True
    except Exception:
        return False

# Use:
sig = sign(b"the order is paid")
print(verify(public_key, sig, b"the order is paid"))    # True
print(verify(public_key, sig, b"tampered"))             # False
print(verify(public_key, sig + b"x", b"the order is paid"))  # False

# Why Ed25519 over RSA for new code:
#   - No key-size choice (always 32 bytes vs RSA's 2048/3072/4096)
#   - No padding choice (vs RSA-PSS vs RSA-PKCS1v15 footguns)
#   - No hash choice (built into the algorithm; SHA-512)
#   - ~50× faster signing, ~10× faster verifying
#   - Smaller signatures (64 bytes vs RSA's 256-512 bytes)
#   - No e=65537 vs e=3 nonsense
#
# Use RSA when:
#   - JWT with RS256 alg (no EdDSA-only verifier on the consumer)
#   - X.509 certificate / TLS scenarios where ECDSA isn't supported
#   - Smart cards / hardware tokens that only support RSA
```

## Example — Junior (Intermediate)

```python
# === JUNIOR EXAMPLE ===
# TASK      - SAME — but persist keys via PEM serialization, support
#             multiple active key versions for rotation, fall back
#             to RSA for legacy.
# APPROACH  - PEM-encode private + public keys for disk/secrets manager
#             storage; "kid" prefix on signatures lets verifiers pick
#             the right public key; RSA path for systems requiring it.
# STRENGTHS- Keys survive process restart; rotation lets you publish
#             a new public key alongside the old one for a transition
#             window.
# WEAKNESSES- PEM is a textual format — keep private PEMs in secrets
#             managers, never alongside source.
from cryptography.hazmat.primitives.asymmetric.ed25519 import Ed25519PrivateKey, Ed25519PublicKey
from cryptography.hazmat.primitives.asymmetric.rsa import generate_private_key as rsa_generate, RSAPrivateKey
from cryptography.hazmat.primitives.asymmetric import padding
from cryptography.hazmat.primitives import serialization, hashes

# 1) Generate + serialize Ed25519.
def gen_and_serialize_ed25519() -> tuple[bytes, bytes]:
    sk = Ed25519PrivateKey.generate()
    pk = sk.public_key()
    sk_pem = sk.private_bytes(
        encoding=serialization.Encoding.PEM,
        format=serialization.PrivateFormat.PKCS8,
        encryption_algorithm=serialization.NoEncryption(),  # in prod: BestAvailableEncryption(passphrase)
    )
    pk_pem = pk.public_bytes(
        encoding=serialization.Encoding.PEM,
        format=serialization.PublicFormat.SubjectPublicKeyInfo,
    )
    return (sk_pem, pk_pem)

def load_ed25519_private(pem: bytes, *, passphrase: bytes | None = None) -> Ed25519PrivateKey:
    return serialization.load_pem_private_key(pem, password=passphrase)

def load_ed25519_public(pem: bytes) -> Ed25519PublicKey:
    return serialization.load_pem_public_key(pem)

# 2) Multi-key registry for rotation.
class KeyRing:
    """Maps kid -> public key. Verifiers consult this; signers pick CURRENT_KID."""
    def __init__(self):
        self._private_keys: dict[str, Ed25519PrivateKey] = {}
        self._public_keys: dict[str, Ed25519PublicKey] = {}
        self.current_kid: str = ""

    def add(self, kid: str, *, private: Ed25519PrivateKey, is_current: bool = False):
        self._private_keys[kid] = private
        self._public_keys[kid] = private.public_key()
        if is_current:
            self.current_kid = kid

    def add_public_only(self, kid: str, public: Ed25519PublicKey):
        self._public_keys[kid] = public

    def sign(self, data: bytes) -> tuple[str, bytes]:
        sk = self._private_keys[self.current_kid]
        return (self.current_kid, sk.sign(data))

    def verify(self, kid: str, sig: bytes, data: bytes) -> bool:
        pk = self._public_keys.get(kid)
        if pk is None: return False
        try:
            pk.verify(sig, data)
            return True
        except Exception:
            return False

# 3) RSA — when you need it (legacy systems, JWT RS256 with old verifiers).
def gen_rsa() -> RSAPrivateKey:
    return rsa_generate(public_exponent=65537, key_size=2048)

def rsa_sign_pss(sk: RSAPrivateKey, data: bytes) -> bytes:
    """Always use RSA-PSS for new RSA code; never PKCS1v15 (broken padding)."""
    return sk.sign(
        data,
        padding.PSS(
            mgf=padding.MGF1(hashes.SHA256()),
            salt_length=padding.PSS.MAX_LENGTH,
        ),
        hashes.SHA256(),
    )

def rsa_verify_pss(pk, sig: bytes, data: bytes) -> bool:
    try:
        pk.verify(
            sig, data,
            padding.PSS(mgf=padding.MGF1(hashes.SHA256()), salt_length=padding.PSS.MAX_LENGTH),
            hashes.SHA256(),
        )
        return True
    except Exception:
        return False

# Usage example:
ring = KeyRing()
ring.add("v1", private=Ed25519PrivateKey.generate(), is_current=False)
ring.add("v2", private=Ed25519PrivateKey.generate(), is_current=True)

kid, sig = ring.sign(b"release artifact contents")
print(kid, sig.hex()[:32])
print(ring.verify(kid, sig, b"release artifact contents"))     # True
print(ring.verify("v1", sig, b"release artifact contents"))    # False (wrong key)
```

## Example — Senior (Production)

```python
# === SENIOR EXAMPLE ===
# TASK      - SAME — production: serve public keys via a JWKS endpoint
#             so verifiers can fetch + cache them; sign via KMS so the
#             private key never lives in application memory; ephemeral
#             keys for short-lived signatures (sigstore-style).
# APPROACH  - JWKS (RFC 7517): JSON document at /.well-known/jwks.json
#             listing public keys with kids; verifiers cache it.
#             Signing happens in KMS (no private-key exposure); the
#             code only sees the signature.
# STRENGTHS - Private key never in app process; rotation via KMS
#             rotation; verifiers auto-discover new keys via JWKS;
#             ephemeral keys (sigstore) eliminate long-term key
#             management entirely for some workflows.
# WEAKNESSES- JWKS caching makes rotation slower (cache TTL bounds
#             how fast a compromised key stops being trusted);
#             KMS signing has API latency — cache when possible.
import os, json, time
import boto3
from cryptography.hazmat.primitives.asymmetric.ed25519 import Ed25519PublicKey
from cryptography.hazmat.primitives import serialization
import httpx

# 1) JWKS endpoint — serve public keys for verifiers.
class JWKSHandler:
    def __init__(self):
        # In prod: load from your secrets manager / config service.
        self.keys: list[dict] = []                    # JWKS-format dicts

    def add_ed25519(self, kid: str, public: Ed25519PublicKey, *, alg: str = "EdDSA"):
        raw = public.public_bytes(
            encoding=serialization.Encoding.Raw,
            format=serialization.PublicFormat.Raw,
        )
        import base64
        x_b64 = base64.urlsafe_b64encode(raw).rstrip(b"=").decode()
        self.keys.append({"kty": "OKP", "crv": "Ed25519", "kid": kid,
                          "alg": alg, "use": "sig", "x": x_b64})

    def jwks(self) -> dict:
        return {"keys": self.keys}

# Serve at /.well-known/jwks.json from your web framework.
# Verifiers fetch it on first verify; cache for jwks-cache-ttl seconds.

# 2) KMS-backed signer — private key never enters the app.
kms = boto3.client("kms")
KEY_ARN = os.environ["KMS_SIGN_KEY_ARN"]               # an EdDSA-supporting KMS key

def sign_via_kms(data: bytes, *, key_arn: str = KEY_ARN) -> tuple[str, bytes]:
    """Returns (kid, signature). The private key never touches us."""
    resp = kms.sign(
        KeyId=key_arn,
        Message=data,
        MessageType="RAW",
        SigningAlgorithm="ECDSA_SHA_256",              # or RSASSA_PSS_SHA_256, depending on key
    )
    # Extract kid from KeyId metadata.
    kid = key_arn.split("/")[-1]
    return (kid, resp["Signature"])

# 3) Verifier with JWKS cache.
class JWKSVerifier:
    def __init__(self, jwks_url: str, *, cache_ttl_s: int = 600):
        self.jwks_url = jwks_url
        self.cache_ttl_s = cache_ttl_s
        self._cache: dict[str, Ed25519PublicKey] = {}
        self._fetched_at: float = 0

    def _refresh(self):
        if time.time() - self._fetched_at < self.cache_ttl_s and self._cache:
            return
        resp = httpx.get(self.jwks_url, timeout=5.0).json()
        new_cache: dict[str, Ed25519PublicKey] = {}
        for jwk in resp.get("keys", []):
            if jwk.get("kty") != "OKP" or jwk.get("crv") != "Ed25519":
                continue
            import base64
            raw = base64.urlsafe_b64decode(jwk["x"] + "==")
            new_cache[jwk["kid"]] = Ed25519PublicKey.from_public_bytes(raw)
        self._cache = new_cache
        self._fetched_at = time.time()

    def verify(self, kid: str, sig: bytes, data: bytes) -> bool:
        self._refresh()
        pk = self._cache.get(kid)
        if pk is None:                                 # unknown kid — refetch immediately
            self._fetched_at = 0
            self._refresh()
            pk = self._cache.get(kid)
            if pk is None:
                return False
        try:
            pk.verify(sig, data)
            return True
        except Exception:
            return False

# 4) Ephemeral keys (sigstore pattern) — generate, sign, throw away.
#    Pair with a transparency log so verifiers can audit "this key
#    existed at this time and signed this artifact". Eliminates the
#    long-term key management problem for one-shot signatures.
def sigstore_style_sign(data: bytes) -> tuple[bytes, bytes, bytes]:
    """Generate ephemeral Ed25519, sign, return (public_key_pem, sig, data).
    Verifier needs the public key + a transparency-log proof to trust it."""
    from cryptography.hazmat.primitives.asymmetric.ed25519 import Ed25519PrivateKey
    sk = Ed25519PrivateKey.generate()
    pk_pem = sk.public_key().public_bytes(
        encoding=serialization.Encoding.PEM,
        format=serialization.PublicFormat.SubjectPublicKeyInfo,
    )
    sig = sk.sign(data)
    # Record (pk, sig, hash(data), timestamp) in a Rekor-style log here.
    return (pk_pem, sig, data)

# Decision rule:
#   sign for a peer with a long-term key  -> Ed25519 (or RSA for legacy)
#   verify with public key only            -> Ed25519PublicKey.from_public_bytes
#   distribute public keys                 -> JWKS endpoint at /.well-known/jwks.json
#   private key must never leave HSM       -> KMS / HSM signing API
#   short-lived per-message signatures     -> sigstore (ephemeral keys + transparency log)
#   JWT signing for OIDC/auth servers      -> RSA RS256 OR Ed25519 EdDSA
#   X.509 cert (TLS, S/MIME)               -> RSA or ECDSA — Ed25519 in TLS 1.3+ only
#   key exchange (Diffie-Hellman)          -> X25519 (NOT Ed25519 — different primitive)
#   need to authenticate WITHOUT signing   -> HMAC (shared secret; previous entry)
#   verifier discovers new keys auto       -> JWKS + cache TTL bounded
#   compromise revocation                  -> rotate kid; bound by JWKS cache TTL
#
# Anti-pattern: using RSA-PKCS1v15 padding for new code. PKCS1v15
# padding has a long history of attacks (Bleichenbacher, etc.) that
# RSA-PSS doesn't share. Many libraries default to PKCS1v15 because
# legacy systems require it. For new code: use Ed25519 (no padding
# choice) OR RSA-PSS with MGF1+SHA256 + max-length salt. NEVER
# PKCS1v15 unless you're talking to a system that requires it.
```

## Anti-Pattern

> [!warning] Anti-pattern
> using RSA-PKCS1v15 padding for new code. PKCS1v15
> padding has a long history of attacks (Bleichenbacher, etc.) that
> RSA-PSS doesn't share. Many libraries default to PKCS1v15 because
> legacy systems require it. For new code: use Ed25519 (no padding
> choice) OR RSA-PSS with MGF1+SHA256 + max-length salt. NEVER
> PKCS1v15 unless you're talking to a system that requires it.

## Tips

- Ed25519 is the default for new asymmetric signing — 32-byte keys, 64-byte signatures, no parameter footguns. Use RSA only when a library or spec requires it (JWT RS256 verifiers, X.509 certs, smart cards).
- For RSA, ALWAYS use RSA-PSS padding with MGF1+SHA256, not PKCS1v15. PKCS1v15 has a long history of padding-oracle attacks; PSS is the modern replacement.
- PEM-encode keys for storage. Private keys go in `PrivateFormat.PKCS8` with `BestAvailableEncryption(passphrase)`; public keys in `PublicFormat.SubjectPublicKeyInfo`.
- For key rotation, prepend a `kid` (key id) to your signatures so verifiers can pick the right public key. Pair with a JWKS endpoint serving all currently-trusted public keys.
- For high-value keys, sign via KMS — the private key never leaves the HSM. The application code only sees the signature; rotation happens in KMS.
- X25519 (key exchange / Diffie-Hellman) is NOT the same primitive as Ed25519 (signing) — though both use Curve25519. Don't confuse them; libraries expose them as separate classes.

## Common Mistake

> [!warning] Using RSA-PKCS1v15 padding for new code. PKCS1v15 has a long history of attacks (Bleichenbacher, Manger) that RSA-PSS doesn't share. Many libraries default to PKCS1v15 because legacy systems require it. For new code, use Ed25519 (no padding choice at all) OR RSA-PSS with `MGF1(SHA256)` + `salt_length=PSS.MAX_LENGTH`.

## Shorthand (Junior → Senior)

**Junior:**
```python
# RSA with PKCS1v15 padding — legacy footgun
sk.sign(data, padding.PKCS1v15(), hashes.SHA256())
```

**Senior:**
```python
# Ed25519 (no padding choice) OR RSA-PSS for new code
Ed25519PrivateKey.generate().sign(data)
# OR:
sk.sign(data, padding.PSS(mgf=padding.MGF1(hashes.SHA256()),
                          salt_length=padding.PSS.MAX_LENGTH),
        hashes.SHA256())
```
